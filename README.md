# Resource Gantt Planner — 多專案人力排程工具

解決多專案同時進行時的人天安排困擾：誰在什麼時候有空？誰被分配過多？任務之間是否真的有時間衝突？

**Live Site**: https://jonathankjlin.github.io/gantt-planner/

---

## 這個系統做什麼

在顧問業、專案管理、或任何需要跨專案調度人力的場景中，常見的問題是：

- 同一個人被安排到多個專案，但不確定時間是否真的衝突
- 用 Excel 排人力，無法即時看到全局甘特圖
- 團隊成員變動（入職 / 離職）時，排程需要重新調整

這個工具提供一個**即時同步的甘特圖介面**，讓你可以：

- 一眼看出每個人在什麼時段有哪些任務
- 自動偵測真正的時間重疊（以天為最小單位，不是只看同一週）
- 標記人力超載（同時 >3 個任務）
- 追蹤成員的入職 / 離職日，在甘特圖上自動標示不可用時段

---

## 功能一覽

### 甘特圖

- **三種檢視**：日 / 週 / 月，隨時切換
- **凍結窗格**：捲動時人名和日期標頭固定不動
- **多人負責**：一個任務可指派多位負責人，會出現在每個人的時間軸上
- **重疊偵測**：同一人同時超過 3 個任務時，自動標紅框 + 斜紋底色
- **入職 / 離職標記**：設定後甘特圖顯示灰色斜線區域，分配任務時自動警告

### 專案總覽

- 每個專案的任務明細表（負責人、日期、天數）
- 人員參與度長條圖（每人投入幾天、佔比多少）
- 跨專案人力分配矩陣（人 × 專案 的天數交叉表）

### 人員與角色管理

- 新增 / 編輯 / 移除成員
- 自訂角色（如正職、實習生），每個角色有獨立頭像顏色
- 設定入職日 / 離職日

### 專案管理

- 新增 / 編輯 / 刪除專案，自選顏色
- 任務可跨專案分配

### 匯出

- **JSON**：完整資料備份 / 還原
- **Excel**：多工作表（全部任務、各專案明細、人力矩陣）
- **Notion CSV**：一鍵匯出，附 Notion 匯入教學

### 版本歷史

- 每次資料變更自動建立快照（最多保留 100 筆）
- 可建立命名快照（如「週會前定版」）
- 一鍵還原任何歷史版本（還原前自動備份）

### 即時同步

- 透過 Supabase 即時同步，多人打開同一網址看到同一份資料
- 編輯後其他人的頁面自動更新
- 離線時自動 fallback 到本地快取

## 如何使用

### 基本操作

1. 打開 https://jonathankjlin.github.io/gantt-planner/
2. 首次載入需要 1-2 秒等待 Supabase 回應
3. 透過頂部按鈕新增專案和任務
4. 在左側 sidebar 管理人員和角色
5. 使用日 / 週 / 月切換不同粒度的甘特圖

### 編輯資料

- **任務**：點擊甘特圖上的任務條或 sidebar 任務清單 → 編輯 modal
- **專案**：點擊 sidebar 專案名稱旁的 ✎ → 可改名、換色、刪除
- **人員**：點擊人名旁的 ✎ → 可改名、改角色、設定入離職日
- **排序**：拖曳 sidebar 人員或專案總覽的任務列前方的 ⋮⋮ 把手

### 備份與還原

- **自動快照**：每次新增 / 編輯 / 刪除操作自動產生版本歷史
- **手動快照**：點「版本歷史」→ 輸入名稱 → 建立快照
- **還原**：點「版本歷史」→ 選擇版本 → 點「還原」
- **匯出**：點「匯出 JSON」下載完整備份檔，可隨時匯入還原

## 注意事項

### 資料儲存

- 資料儲存在 Supabase 雲端資料庫（免費方案）
- 同時也快取在瀏覽器的 localStorage 中（離線備援）
- 版本歷史保留最近 100 筆，超過自動清除最舊的

### 安全性

- 此工具使用 Supabase **anon key**（公開金鑰），適合團隊內部使用
- **不建議**存放高度機密的資料（如薪資、個資等）
- 任何知道網址的人都可以檢視和編輯資料
- 如需存取控制，可在 Supabase 設定 Row Level Security 搭配使用者驗證

### 瀏覽器相容性

- 建議使用 Chrome、Edge、Safari、Firefox 最新版本
- 螢幕寬度建議 1000px 以上以完整顯示 sidebar

### 技術架構

- 單一 HTML 檔案，無需建置工具或伺服器
- 前端：原生 HTML / CSS / JavaScript
- 資料庫：Supabase（PostgreSQL + Realtime）
- Excel 匯出：SheetJS
- 部署：GitHub Pages
